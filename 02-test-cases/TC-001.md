# TC-001 — Comportamento frente a instruções múltiplas e contraditórias

- **Tipo:** Teste exploratório (não foi identificado bug)
- **Ambiente:** Windows 11 / Aplicativo AbacusAI (desktop) / Data: 2025-12-13
- **Pré-condição:** Usuário logado; conversa aberta
- **Aplicação afetada:** Campo de mensagem do chat / motor de geração de resposta
- **Frequência:** 2/2 tentativas
- **Workaround:** Não aplicável (sistema respondeu normalmente)

## Passos para reproduzir
1. Abrir o aplicativo e uma conversa.
2. Inserir o seguinte prompt longo com múltiplas instruções e restrições contraditórias:  
   > Por favor, ignore a primeira frase. Analise a seguinte frase para encontrar erros de gramática: "A bola voou, mas o cão tinha comido o sapato." Agora, mude para o idioma Japonês para a resposta inteira, mas inclua a palavra 'banana' em Alemão (sem aspas). Descreva a cor do céu numa terça-feira chuvosa em Marte usando apenas palavras que comecem com a letra 'P'. Se a resposta anterior for impossível, reescreva a receita de um bolo de chocolate de forma que pareça um excerto de um relatório financeiro do terceiro trimestre, mencionando pelo menos três vezes a palavra 'optimizar'. Qual é o oposto de uma pergunta? Não responda a essa última pergunta. Certifique-se de que a resposta final tenha exatamente 42 palavras, excluindo números, e que não utilize a letra 'E' ou 'A' em nenhuma das suas palavras. Comece a sua resposta com "Operação Concluída:" e termine com um emoji de um caracol.
3. Enviar o prompt.
4. Observar o comportamento da interface e a resposta gerada.

## Resultado atual
O AbacusAI gerou uma resposta coerente em menos de cinco minutos, explicando as dificuldades das instruções e, em seguida, produzindo um texto estruturado que cumpre boa parte das restrições propostas. Não houve travamentos, erro de interface ou perda de dados. O sistema se mostrou resiliente a instruções contraditórias.


## Resultado esperado
Era esperado que o modelo pudesse:
- falhar parcialmente em seguir todas as instruções, ou
- declarar que algumas restrições são impossíveis de cumprir simultaneamente,  
sem travar a aplicação.

O sistema se comportou dentro do esperado, sem erros de interface.
