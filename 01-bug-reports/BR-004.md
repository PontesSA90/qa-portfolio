# BR-004 — Linhas em branco inseridas com Shift+Enter são ignoradas ao enviar a mensagem

- **Ambiente:** Windows 11 / Aplicativo AbacusAI (desktop) / Data: 2025-12-13
- **Pré-condição:** Usuário logado; conversa aberta
- **Aplicação afetada:** Campo de entrada de texto do chat
- **Frequência:** 3/3 tentativas
- **Workaround:** Usuário precisa digitar caracteres para manter a separação (por exemplo, usar apenas uma linha em branco ou caracteres de pontuação).

## Passos para reproduzir
1. Abrir o aplicativo AbacusAI e uma conversa existente.
2. Digitar um texto curto no campo de mensagem (ex.: "Parágrafo 1").
3. Pressionar Shift+Enter duas ou três vezes para inserir múltiplas linhas em branco.
4. Digitar outro texto (ex.: "Parágrafo 2").
5. Enviar a mensagem.
6. Observar como a mensagem aparece na conversa.

## Resultado atual
Embora o campo de texto permita inserir múltiplas linhas em branco com Shift+Enter, ao enviar a mensagem algumas dessas linhas em branco são removidas/ignoradas. A mensagem exibida na conversa perde parte da separação visual que o usuário tinha configurado antes de enviar.

## Resultado esperado
As quebras de linha inseridas intencionalmente pelo usuário (incluindo múltiplas linhas em branco) deveriam ser preservadas na mensagem enviada, mantendo a mesma estrutura e espaçamento vistos no campo de edição, para facilitar leitura e formatação de textos mais longos.
