# TC-001 — Behavior under multiple and contradictory instructions

- **Type:** Exploratory test (no bug identified)  
- **Environment:** Windows 11 / AbacusAI Desktop App / Date: 2025-12-13  
- **Pre-condition:** User is logged in; a conversation is open  
- **Affected area:** Chat message input / response generation engine  
- **Frequency:** 2/2 attempts  
- **Workaround:** Not applicable (system responded normally)

## Test steps

1. Open the AbacusAI desktop application and open a conversation.  
2. Enter the following long prompt containing multiple and contradictory instructions and restrictions:  

   > Por favor, ignore a primeira frase. Analise a seguinte frase para encontrar erros de gramática: "A bola voou, mas o cão tinha comido o sapato." Agora, mude para o idioma Japonês para a resposta inteira, mas inclua a palavra 'banana' em Alemão (sem aspas). Descreva a cor do céu numa terça-feira chuvosa em Marte usando apenas palavras que comecem com a letra 'P'. Se a resposta anterior for impossível, reescreva a receita de um bolo de chocolate de forma que pareça um excerto de um relatório financeiro do terceiro trimestre, mencionando pelo menos três vezes a palavra 'optimizar'. Qual é o oposto de uma pergunta? Não responda a essa última pergunta. Certifique-se de que a resposta final tenha exatamente 42 palavras, excluindo números, e que não utilize a letra 'E' ou 'A' em nenhuma das suas palavras. Comece a sua resposta com "Operação Concluída:" e termine com um emoji de um caracol.

3. Send the prompt.  
4. Observe the UI behavior and the generated response.

## Actual result

AbacusAI generated a coherent response in less than five minutes, explaining the difficulties of the instructions and then producing a structured text that satisfies a significant part of the proposed constraints.  
There were no freezes, UI errors, or data loss. The system showed resilience when handling contradictory instructions.

## Expected result

It was expected that the model would:

- partially fail to follow all instructions, or  
- explicitly state that some constraints are impossible to satisfy simultaneously,  

without causing the application to freeze or crash.

The system behaved as expected, with no UI errors.
